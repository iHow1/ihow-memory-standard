# Contributing

iHow Memory v0.1 is an open-standard draft. The most useful contributions at this stage are improvements to terminology, reliability scenarios, protocol boundaries, conformance criteria, and documentation clarity.

## Good First Contributions

- Clarify ambiguous protocol terms.
- Add missing failure modes to reliability scenarios.
- Suggest additional conformance checks.
- Improve bilingual wording.
- Point out privacy, safety, audit, or namespace boundary risks.

## Out of Scope for v0.1

- tool integration code
- SDKs
- runtime services
- deployment recipes
- hosted service code
- customer-specific workflows
