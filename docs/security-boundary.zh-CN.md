# 安全边界

iHow Memory 的设计目标不是无限制回忆，而是有范围、可审计的记忆。

## 命名空间隔离

兼容系统应该在读取、写回生效或审计暴露之前执行隔离：

- tenant scope
- customer scope
- project scope
- user scope

除非策略明确允许，并且审计记录说明原因，否则一个范围内的记忆不应该泄漏到另一个范围。

## 敏感数据

公开示例应避免密码、凭证、私有账号数据、私有客户数据、原始私聊和运维内部细节。

实现该协议的系统应该支持脱敏、审核队列、范围检查、审计记录、删除和回滚。

## v0.1 边界

本仓库只在规格层描述安全边界，不发布运行时内部细节或私有运维流程。
